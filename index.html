    最近十分火爆的社交游戏TwitFi怎么玩？
    原文标题：《TwitFi：基于 Twitter，一种将社交与游戏结合的新型 GameFi》 原文作者：0xBun，深潮 TechFlow 尽管 X2E 已经落幕，但它仍然是 Web3 中相对新颖的体系之一，吸引着不少流动性。X2E 是一种高灵活性的叙事，其表现形式取决于人们的想象力。也就是说「X」可以披上人们想要的任何外衣，如 Play、Write 甚至是 Sleep。今天我们将要看到的「X」是 Tweet，看它是如何通过发布推文赚取奖励的。 什么是 TwitFi？ TwitFi 是一款面向 Web 3 社交网络游戏，它具有 GameFi 功能。游戏上手很简单，只需将你的钱包和推特账户连接至它们的官网，游戏的准备步骤就完成了三分之二。 最后，你需要准备 TwitFi 的 NFT，它是以鸟为基础设计的交互 NFT。玩家可以在市场或 Mint 期间，花钱或免费 Mint 一只宠物蛋。当然，不同种类的蛋所孵化出来的鸟的种类也不同。例如，如果是免费的蛋，那孵化出来的多半是 Baby 类型的鸟。需要注意的是，Baby 类型的鸟是无法在二级市场出售的。 准备好了这些，玩家便可以在推特上发布带 #TwitFi 的推文来赚取游戏内的Token了， Token 可在游戏中使用或兑现。 T2E 机制 如果只是发布推文赚取奖励，那未免过于无趣，也很难抵御投机者的攻击。为了让 TwitFi 看起来更像游戏，项目决定在 NFT 上做文章。 如前所述，当你将蛋孵化成鸟的时候，鸟会有四种类型，以及五个等级。类型和等级不同的鸟，每条推文赚取的 Token 也不相同。 并且，NFT 并不能直接赚取 Token ，而是要通过一种名为「饲料」的媒介。当我们的饲料用完时，便再也无法通过发布推文来赚取 Token 。后续则需要等待饲料的重新补充。鸟的等级不同，饲料的恢复速度也不一样，如下图所示。 但是不管等级是多少，每个 NFT 所能拥有的饲料上限为 10 个，玩家可以通过购买饲料箱来积累最多 50 个饲料。 显然，等级的高低是赚取 Token 的关键，那么如何提升鸟的等级呢？除了在二级市场购买高等级鸟之外，玩家可以通过消耗赚取的 Token （TWT）来提升级别，并且在提升等级之前，还有一段等待期： 为了进一步削减 TWT 的市场抛压，和提高游戏的生命力，NFT 中还引入了「病毒」机制。当你发推文时，NFT 有一定的概率感染病毒。如果 NFT 被感染，你发推不会得到 TWT。感染会在 7 天后痊愈。或者，你可以在市场上购买药物来立即治愈它。 感染概率取决于 NFT 的推文数量： 需要注意的是，虽然感染之后会痊愈，但是在感染次数满 20 次之后，NFT 会自动销毁。 游戏的优缺点 TwitFi 是一款主打社交的游戏，它的目标人群明确：经常使用社交软件的人。TwitFi 的独特之处在于，它允许玩家通过在社交媒体上发布推文来赚取真实收益。这使得玩家在享受互动的同时还能获得经济利益。 另外，游戏中宠物的设定增加了游戏的可玩性。这是因为宠物作为游戏中的一个元素，是增加游戏趣味性和复杂性的一种手段——TwitFi 通过宠物衍生出来的一系列机制，赋予了游戏可持续运行的动力。 然而，游戏的优点也可能是它的缺点。例如，游戏中提供的「饲料」恢复速度取决于鸟的等级，这可能会导致玩家觉得购买高等级 NFT 是必要的。以及游戏在可玩性方面表现较差，单一的发推并不能产生多大的用户黏度，特别是熊市期间。 总之，TwitFi 目前还处于早期，通过推文和 NFT 的方式，打通了 Web2 与 Web3 的桥梁，给 Web3 带来新的血液，游戏的创意和想法值得肯定。但其护城河尚未建立，模式容易被复刻，以及正式发行后，游戏是否能够稳定地运营持续下去还有待商榷。 原文链接 律动 BlockBeats 提醒，根据银保监会等五部门于 2018 年 8 月发布《关于防范以「虚拟货币」「区块链」名义进行非法集资的风险提示》的文件，请广大公众理性看待区块链，不要盲目相信天花乱坠的承诺，树立正确的货币观念和投资理念，切实提高风险意识；对发现的违法犯罪线索，可积极向有关部门举报反映。

    原文来源：yyy，Web3caff研究员 以太坊二层扩容方案 @optimismFN 预计在 2023 年实现 Bedrock 版本的升级，近期已完成测试网 Goerli 的 Bedrock 升级，主网升级仅一步之遥。 Optimism 团队也把此次升级作为当前最重要的核心目标，具有一定的战略意义。 此条推文简单聊聊什么是 Bedrock 以及为什么要进行 Bedrock 升级。 Rollup 方案的困局 目前 Rollup 赛道已经有不少项目上线主网（如 Arbitrum、ZkSyncMetisDAO），但它们都存在一些不可忽略的问题： 1) EVM 兼容性；虽然一些基于 OP 的 Rollup 方案实现了 EVM 等效性，但 EVM 兼容/等效并不意味着项目能够无缝迁移，项目方仍然需要进行或多或少的代码调整。 网络交易费用仍然偏高；Rollup 的费用主要分为 2 部分：一是自身的运营费用；二是占用 L1 区块空间的数据可用性（DA）费用，其中 DA 的费用占大头。 执行层将相关证明以 calldata 的函数形式上传到 L1，在一定程度上占用了主网的区块空间。单一定序器的中心化问题。目前大多数 Rollup 解决方案的定序器都是由项目方自身或者单一的运营商运行，存在严重的中心化问题。 如 Arbitrum 在今年 1 月份发生的宕机事故，导致网络在数小时内无法正常出块。究其原因，问题就发生在 Arbitrum 官方运行的定序器上。 @optimismFND Bedrock 升级正是旨在解决上述问题。Bedrock 升级后会发生如下重大变化： 1) L1 到 L2 的存款时间缩减 4 倍以上（10min 缩短至 2.5min）； 2) 向 L1 提交数据的成本削减约 20%； 3) 与 Geth 的差异精简到约 500 行代码； 4) 支持多样化的替代证明系统（包括 ZK）。 为了实现更低的成本，@optimismFND 开发了 Cannon, 一个全新的交互式欺诈证明系统。 Optimism 现在采用的是单轮交互欺诈证明解决争议，为什么多轮交互反而能降低成本呢？ 很容易理解，交互在链下进行，不断缩小争议的范围，然后再提交主网做验证。 以太坊等效性 @optimismFND 早在去年年底就实现了 EVM 等效性，为了更大程度地兼容以太坊，Bedrock 升级提出了以太坊等效性。 简单来讲，就是让 Arbitrum 上的代码和以太坊上的代码差异最小化，让以太坊上的原生项目以极低的成本迁移过来。 多客户端系统 @optimismFND 的 Bedrock 升级将建立一个多客户端生态系统，激励外部团队创建其他客户端。 通过多客户端的形式实现去中心化，从而大幅降低了网络的单点风险。 以上就是关于 Arbitrum Bedrock 升级的介绍。虽然目前 Optimism 团队的重心是 Bedrock 升级，但其最终的愿景是旨在利用 OP Stack 实现可互操作的「超级链」生态。  原文链接 律动 BlockBeats 提醒，根据银保监会等五部门于 2018 年 8 月发布《关于防范以「虚拟货币」「区块链」名义进行非法集资的风险提示》的文件，请广大公众理性看待区块链，不要盲目相信天花乱坠的承诺，树立正确的货币观念和投资理念，切实提高风险意识；对发现的违法犯罪线索，可积极向有关部门举报反映。
