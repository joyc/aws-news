    比特币NFT「开荒进行时」：Ordinals协议、FOMO和争论
    1 月 21 日，比特币核心贡献者 Casey Rodarmor 创建的 Ordinals 协议发布了 0.4.0 版本，正式将 NFT 带入了比特币主网。至今，已有约 9000 个 NFT 通过 Ordinals 协议在比特币主网上被 Mint 出来。 第 9000 个通过 Ordinals 协议在比特币主网上被 Mint 出来的 NFT Ordinals 协议有一个支持跟踪和传输单个比特币最小单位 Satoshi 的编号方案，将不同的 Satoshi 按照被开采的顺序编号，并以先进先出的顺序从交易输入转移到交易输出。 每个 Satoshi 都有了独特的编号作为身份标识后，就可以被单独跟踪或交易。在此基础上，再往每个 Satoshi 里写入数据（如图像、文本、SVG、HTML 等），就完成了将 Satoshi 从一个「比特币最小单位」向一个「比特币 NFT」的转变。 这些比特币 NFT 不支持链下数据的引用。也就是说，所有比特币 NFT 包含的数据完全存储在链上，不会因集中式的链下存储出现问题而造成丢失，同时也无法更改。 与以太坊上普遍使用成熟的中心化 NFT 交易市场不同，在比特币主网上，一切都显得那么原始，犹如「开荒」——没有 NFT 交易市场，一切全靠 OTC。没有 Mint 页面，发售模式大部分是「向创作者打钱->创作者把 NFT 上链->手动向购买者分发 NFT」。Ordinals 甚至都还没来得及出一个轻量化的钱包应用，若想要将 NFT 上链或可靠地将 NFT 转给其他地址，必须安装好比特币核心钱包并通过命令行来使用 Ordinals 的完整功能。 总结一下参与 Ordinals 协议开启的「比特币主网 NFT」热潮的要点： - 如果只是购买并收藏 NFT，只需一个比特币钱包（推荐使用 P2TR 地址类型） - 如果在接收到 NFT 后希望转手/自己作为创作者发售 NFT，需要安装比特币核心钱包，并通过命令行与 Oridinals 交互（官方教程） - 如果要查阅自己所有的 NFT，需要根据 NFT（Satoshi） 编号，在Ordinals Explorer上搜索监视。 尽管使用上如此不友好，但 FOMO 情绪却似乎没受到多少影响： 知名 NFT 项目 Forgotten Runes Wizard's Cult 的 CEO @dotta 花 0.2 BTC 买了块石头。目前，在该项目 Discord 内，求购 offer 已经给出 1 BTC 以上的价格 Ordinal Punk 以 0.01 BTC 的价格发售 100 个，最新成交价已经达到 1.9269 BTC 争议伴随着 FOMO，尤其是「Taproot Wizard」这个 NFT 项目引起的巨大争论。Taproot Wizard 用一张 3.94 MB 的图片几乎把整个区块塞满，使比特币网络挖出了历史最大区块。 比特币网络挖出了历史最大区块。2017 年的 SegWit 升级与 2021 年的 Taproot 升级，将单个区块最大规模提升到了 4MB，为如今的 Ordinals 铺平了道路 创造历史的「Taproot Wizard #0001 'Bald Wizard'」 另一位比特币核心开发者 Luke Dashjr 直接把这笔交易称为「垃圾邮件」并快速开发了一款「垃圾邮件过滤器」，用于供矿工筛选并过滤包含 Ordinals 协议创造的比特币 NFT 的交易广播。这个举动在是否支持比特币非金融应用上又增添了一个吵架点——NFT 是不是「垃圾邮件」暂且两说，比特币人在「提倡审查」这个问题上又吵了起来。不过，正是「聪明脑袋」们的不停争论，使比特币社区乃至比特币本身蓬勃发展，直到今天。 Ordinals 协议仍在早期，轻量化钱包应用、NFT 交易市场等「基建」都尚待搭建。在这些「基建」落地后，比特币主网 NFT 市场大概率会再迎来一波增长，因此值得持续关注。此外，Ordinals 协议对比特币网络造成的长期影响到底会是什么，也值得我们思考——是非金融应用对矿工激励的利更大，还是金融应用成本变得更为高昂的弊更大呢？ 无论如何，Ordinals 协议都会继续在比特币主网上存在、发展。比特币 NFT，会是 NFT 市场的下一个大热点吗？让我们拭目以待。
